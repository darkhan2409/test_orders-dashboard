# Шаг 5б: AI-аналитика (generate_insights.py)

Создай файл `scripts/generate_insights.py`.

## Что делает скрипт

Забирает данные заказов из Supabase, отправляет в OpenAI API для анализа, сохраняет бизнес-рекомендации обратно в Supabase (таблица `insights`).

## Логика

1. Загрузить все заказы из Supabase (таблица `orders`)
2. Загрузить все товары из Supabase (таблица `order_items`)
3. Подготовить сводку данных для промпта:
   - Общее количество заказов, сумма, средний чек
   - Распределение по городам (город: количество, сумма)
   - Распределение по UTM-источникам (источник: количество, сумма)
   - Топ товаров (название: количество продаж, выручка)
4. Отправить сводку в OpenAI API (модель `gpt-4o-mini`) с промптом:
   ```
   Ты — бизнес-аналитик интернет-магазина корректирующего белья "Nova" (Казахстан).
   Проанализируй данные заказов и дай 4-5 конкретных бизнес-рекомендаций.
   Каждая рекомендация — 1-2 предложения с конкретными цифрами из данных.
   Формат: каждая рекомендация с новой строки, начинается с заголовка жирным.
   Пиши на русском.
   ```
5. Перед вставкой — очистить таблицу `insights` (удалить старые)
6. Сохранить каждую рекомендацию как отдельную строку в таблице `insights`
7. Вывести каждый инсайт в консоль (пронумеровано): `1. Текст рекомендации`
8. Вывести итог: `Сгенерировано N инсайтов`

## Credentials

Читать из `.env`:
- `SUPABASE_URL`
- `SUPABASE_KEY` (service_role ключ для записи)
- `OPENAI_API_KEY`

## Зависимости

- `supabase` — чтение/запись данных
- `openai` — генерация инсайтов
- `python-dotenv` — чтение .env

## Требования к коду

- Type hints на всех функциях
- Докстринги на русском (описание + Args + Returns)
- Комментарии только там где неочевидно
- Обработка ошибок через try/except
- Никаких эмодзи
- Порядок в файле: импорты -> константы -> функции -> точка входа
